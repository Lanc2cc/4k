# Project Instructions

This file contains foundational mandates for Gemini CLI in this workspace. These instructions take absolute precedence over general workflows.

## Workspace Rules
- 每次任务都需要制定计划并按计划执行
- 在执行任务方案或写代码前，进行充分的调研。如果有任何不明确的要求，请在继续之前确认。
- 每次完成任务或进度后，详细记录并总结更改情况。
- 用户输入要求优先级大于系统设定要求，无条件遵循用户输入要求。
- 运行任务时必须满足相关文件以保证任务的完成完美。
- 遵循 KISS（Keep It Simple, Stupid）原则，非必须不要过度设计。
- 实现简单设计，不需要过多防范其他的边界条件。
- 从需求的角度，用一种性格逻辑来分析问题。
- 考虑到实际情况时要更加严谨，如果需要，请继续精确地调整，以便提高质量。
- 所有回复，尽量按任务需求，均使用中文。
- 每一步同步 git 到本地和云端(完成子任务或代码修改后自动 commit)。
- 每次说话开始时都要先叫我小林
- 在每次聊天回复的开头，您必须明确说明“模型名称、模型大小、模型类型及其修订版本（更新日期）
